# 项目语境

这份文件给后续开发、重构和 AI 协作使用。讨论架构时优先使用这里的名字，避免同一个东西被叫成多种名字。

## 项目定位

chatgpt2api 是一个 ChatGPT Web 上游代理和图片生成控制台，不只是普通聊天前端。它的核心价值是：

- 管理多账号号池。
- 把 OpenAI 兼容接口转成 ChatGPT Web 上游请求。
- 支撑文生图、图生图、轮询、失败重试、图片存储。
- 给管理员提供账号、日志、图片、设置和诊断能力。

## 核心领域词

### 号池

号池是可用于请求上游 ChatGPT Web 的账号集合。号池不只是账号列表，还包含账号状态、额度、并发槽、失败次数、刷新和重登能力。

### 账号

账号是一个可被后端选中的上游身份。关键字段包括 `access_token`、`refresh_token`、`id_token`、`email`、`type`、`status`、`quota`、`proxy`。

### 用户密钥

用户密钥是调用本服务管理接口或兼容接口的本地身份。它不是上游 ChatGPT 账号。

### 图片链路

图片链路指从 `/v1/images/*` 或 `/api/image-tasks/*` 入口，到账号选择、图片上传、上游会话、SSE 解析、图片轮询、下载、日志落盘的完整过程。

### 图片任务

图片任务是新控制台应使用的长任务抽象。浏览器提交任务后通过任务列表轮询状态，避免长请求卡死页面。当前状态包括 `queued`、`running`、`success`、`error`。

### 上游文本回复

上游文本回复指 ChatGPT Web 在图片请求里没有产出图片资产，而是返回一段普通文本。它可能是拒绝说明、任务参数 JSON、排队/状态文本或无法识别的模型回复。前端应该把它当成可诊断失败，而不是简单显示 400。

### 恢复轮询

恢复轮询指图片任务已有 `conversation_id` 且初次等待超时后，再继续向上游查询图片结果。它只适合有会话 ID 的超时类失败。

### 画廊

画廊是已落盘图片的管理页面。它接本地图片索引、缩略图、标签、下载、删除和存储统计，不等同于图片任务队列。

### 运行日志

运行日志指服务进程级输出、后台任务输出和异常堆栈。当前主要可见的是 API 调用日志，运行日志后续可以独立接入。

## 架构约束

- 后端核心图片链路稳定优先，前端重写阶段不大拆。
- 新前端优先接任务接口，不把图片长请求直接绑在浏览器等待里。
- 日志和图片任务都要能解释失败原因，但两者不要混为一个概念。
- 注册机暂时是独立任务域，不进入第一版主控制台。
