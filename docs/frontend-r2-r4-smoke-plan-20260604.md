# R2-R4 前端 Smoke 执行清单

时间：2026-06-04

目标：在不误伤生产账号池、真实图片和远端存储的前提下，验证新 Vue 控制台的写入类、外部副作用类和破坏性动作。

## 前置原则

- 所有测试对象必须提前命名为 `codex-smoke-*` 或标记为“测试”，便于识别和回滚。
- 每次只测一个动作，记录请求结果和页面状态后再进入下一项。
- R2 保存类动作可以在明确测试对象上执行；普通编辑弹窗点击“保存”即视为确认。
- R3 外部副作用动作必须在执行前再次确认目标服务、代理、存储或账号来源。
- R4 破坏性动作默认不执行，除非对象是专门创建的测试数据。

## 需要准备的测试对象

| 对象 | 建议名称 | 用途 | 回滚方式 |
| --- | --- | --- | --- |
| 测试账号 | `codex-smoke-account` | 账号编辑、代理引用、单账号刷新、恢复状态 | 恢复原字段或删除测试账号 |
| 测试图片 | 文件名含 `codex-smoke` | 标签编辑、预览、下载、删除验证 | 重新上传或只删除测试图片 |
| 测试代理分组 | `codex-smoke-proxy` | 代理分组新增、编辑、禁用、删除 | 删除该代理分组 |
| 测试设置项 | 低风险数值字段 | 设置保存、保存后恢复 | 记录原值并改回 |
| 测试日志 | 只选测试请求产生的日志 | 日志删除验证 | 不建议恢复，需明确是测试日志 |

## R2 可恢复写入

| 页面 | 动作 | API | 验证点 | 回滚 |
| --- | --- | --- | --- | --- |
| 账号管理 | 编辑测试账号备注/代理字段 | `POST /api/accounts` | 保存后列表字段更新，刷新页面仍存在 | 改回原字段 |
| 账号管理 | 导出选中测试账号 | `/api/accounts/export` | 弹确认，导出文件数量正确 | 无写入 |
| 图片画廊 | 编辑测试图片标签 | `POST /api/images/tags` | 标签保存后列表和详情一致 | 清空或改回标签 |
| 图片画廊 | 保存保留天数测试值 | `POST /api/settings` | 设置页和画廊页口径一致 | 改回原天数 |
| 代理管理 | 新增/编辑测试代理分组 | `POST /api/proxy/profiles` | 分组列表展示，账号编辑可引用 | 删除测试分组 |
| 设置中心 | 保存低风险设置字段 | `POST /api/settings` | 保存成功后 reload 保持 | 改回原值 |

## R3 外部副作用

| 页面 | 动作 | API | 执行条件 |
| --- | --- | --- | --- |
| 账号管理 | OAuth 登录流程 | OAuth callback | 使用测试 OpenAI/Microsoft 账号 |
| 账号管理 | 远程 CPA 文件读取 | `/api/cpa/*` | 只连测试 CPA 服务器 |
| 账号管理 | Sub2API 分组/账号读取 | `/api/sub2api/*` | 只连测试 Sub2API 服务器 |
| 账号管理 | 刷新选中测试账号额度 | `/api/accounts/refresh*` | 只选测试账号，批量大小按当前默认 |
| 代理管理 | 测试代理连通性 | `/api/proxy/test`、`/api/proxy/profiles/test` | 只测测试代理 |
| 设置中心 | WebDAV/R2/备份测试 | image storage / backup APIs | 只测测试存储或明确可用配置 |

## R4 破坏性动作

| 页面 | 动作 | API | 允许条件 |
| --- | --- | --- | --- |
| 账号管理 | 删除账号 | `DELETE /api/accounts/{id}` | 只删除测试账号 |
| 日志管理 | 删除日志 | `POST /api/logs/delete` | 只删除测试请求日志 |
| 图片画廊 | 删除图片 | `POST /api/images/delete` | 只删除测试图片 |
| 图片画廊 | 清理图库 | `POST /api/images/delete`、cleanup APIs | 默认禁止，除非测试图库 |
| 设置中心 | 删除备份 | `POST /api/backups/delete` | 只删除测试备份 |

## 建议执行顺序

1. 新增测试代理分组 `codex-smoke-proxy`，编辑一次，再删除。
2. 选择测试图片，编辑标签，刷新图库确认，改回标签。
3. 选择测试账号，编辑备注/代理引用，刷新页面确认，改回原值。
4. 只对测试账号执行单账号刷新，确认进度条和结果。
5. 如配置可用，再执行 R3：代理测试、CPA/Sub2API 读取、WebDAV/R2/备份测试。
6. 最后只对测试对象执行 R4 删除验证。
